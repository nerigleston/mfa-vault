How to configure a new site

Guide to finding the URL Pattern and CSS Selector for any MFA login page

What you need to configure

To automatically fill the MFA code on a site, the extension needs two pieces of data:

URL Pattern A substring of the page address where the MFA field appears. Example: github.com
CSS Selector The identifier of the input field where the code will be typed. Example: #app_totp
1 How to find the URL Pattern

The URL Pattern is the part of the page address that identifies the site. The extension checks whether this substring appears in the active tab URL before attempting to fill.

Step by step
1
Navigate to the site's login page until you reach the screen asking for the MFA code.
2
Look at Chrome's address bar and copy the domain (the part after https:// up to the first /).
3
Use that domain as the URL Pattern. The more specific, the better — so the rule doesn't conflict with other pages on the same domain.
Examples
URL in the address bar Suggested URL Pattern
https://github.com/login/two-factor github.com
https://signin.aws.amazon.com/signin?client_id=... signin.aws.amazon.com
https://hub.notro.io/#/login hub.notro.io
https://app.company.com/auth/mfa app.company.com
💡 If the site uses different subdomains (e.g. login.company.com and app.company.com), use the full subdomain to avoid the rule firing in the wrong place.
2 How to find the CSS Selector

The CSS Selector identifies the input field where the MFA code will be typed. You'll use Chrome's DevTools to inspect the element.

Step by step
1
With the MFA field visible on screen, right-click directly on the field and choose "Inspect".
2
DevTools opens with the <input> element highlighted in blue. Look for the id, name, and autocomplete attributes on the highlighted line.
3
Build the selector following the priority order below.
4
Paste the selector into the "CSS Selector" field in the account's ⚙ panel.
Selector priority order
Type Example Stability
ID — id attribute #mfaCode Excellent
Name — name attribute input[name="otp"] Excellent
Autocomplete semantic input[autocomplete="one-time-code"] Good
Placeholder input[placeholder="enter code"] Fair
CSS Class .mfa-input Fragile
How to read the inspected HTML

Example of what you'll see in DevTools:

<input id="mfaCode" ← use #mfaCode name="mfaCode" ← or input[name="mfaCode"] autocomplete="one-time-code" type="text" placeholder="enter code" >

In this case use #mfaCode — the id attribute is the most stable.

3 Single input or split fields?

Some sites use a single field for all 6 digits. Others show 6 separate boxes (one per digit). The extension detects this automatically — but the selector changes.

✓ Single input
<input id="app_totp" maxlength="6">

Selector: #app_totp
Returns 1 element → fills the entire code.

⬡ 6 split fields
<input name="code-0" maxlength="1"> <input name="code-1" maxlength="1"> ... (up to code-5)

Selector: input[name^="code-"]
Returns 6 elements → fills one digit per field.

💡 The ^= in the selector means "starts with". Use it when the fields share a common prefix in their name or id attribute.

5 Alternative: use AI to identify the fields

If you'd rather not inspect manually, copy the HTML of the field and use the prompt below. The AI returns everything ready to paste into the extension.

1 — Copy the MFA field HTML
1
Open the site's MFA page with the code field visible.
2
Right-click on the field (or the area around it) → Inspect.
3
In DevTools, the <input> element is highlighted in blue. Right-click it → Copy → Copy outerHTML.
4
If the field is part of a larger form, you can also copy the parent element (<form> or the containing <div>) to give the AI more context.
2 — Use this prompt
I'm configuring my MFA Vault Chrome extension and need to identify the data for auto-filling the MFA code on this site. Site: [SITE NAME] MFA page URL: [PASTE URL HERE] Field HTML (copied via DevTools → Inspect → Copy outerHTML): [PASTE HTML HERE] Please return exactly in this format: URL Pattern: (substring of the URL that identifies the page, e.g. github.com) CSS Selector: (most stable selector for the input field, prefer id > name > semantic attributes) Type: (Single input OR 6 split fields) autoSubmit: (true if there is a submit button in the same form, false if unsure)
3 — Apply the result in the extension
1
Open MFA Vault and click ⚙ on the account for that site.
2
Click + Add rule (or ↙ Known site if a preset already exists).
3
Fill in URL Pattern and CSS Selector with the values returned by the AI.
4
Check "Submit form after filling" if autoSubmit is true.
5
Click outside to save and test by clicking the code with the site open.
💡 If you see "Field not found — copied", the selector may be wrong or the page hasn't rendered the field yet. Try copying a larger chunk of HTML (the <form> or the parent component) and repeat the process.
