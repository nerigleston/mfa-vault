Como configurar um site novo

Guia para encontrar a URL Pattern e o Seletor CSS de qualquer página de MFA

O que você vai configurar

Para preencher automaticamente o código MFA em um site, a extensão precisa de dois dados:

URL Pattern Um trecho do endereço da página onde o campo MFA aparece. Exemplo: github.com
Seletor CSS O identificador do campo de texto onde o código será digitado. Exemplo: #app_totp
1 Como encontrar a URL Pattern

A URL Pattern é o trecho do endereço da página que identifica o site. A extensão verifica se esse trecho aparece na URL da aba ativa antes de tentar preencher.

Passo a passo
1
Acesse a página de login do site até chegar na tela que pede o código MFA.
2
Olhe a barra de endereços do Chrome e copie o domínio (a parte depois de https:// até a primeira /).
3
Use esse domínio como URL Pattern. Quanto mais específico, melhor — assim a regra não conflita com outros sites do mesmo domínio.
Exemplos
URL na barra URL Pattern sugerida
https://github.com/login/two-factor github.com
https://signin.aws.amazon.com/signin?client_id=... signin.aws.amazon.com
https://hub.notro.io/#/login hub.notro.io
https://app.empresa.com.br/auth/mfa app.empresa.com.br
💡 Se o site usa subdomínios diferentes (ex: login.empresa.com e app.empresa.com), use o subdomínio completo para evitar que a regra dispare no lugar errado.
2 Como encontrar o Seletor CSS

O Seletor CSS identifica o campo de texto onde o código MFA será digitado. Você vai usar o DevTools do Chrome para inspecionar o elemento.

Passo a passo
1
Com o campo MFA visível na tela, clique com o botão direito diretamente no campo e escolha "Inspecionar".
2
O DevTools abre com o elemento <input> destacado em azul. Procure os atributos id, name e autocomplete na linha destacada.
3
Monte o seletor seguindo a ordem de prioridade abaixo.
4
Cole o seletor no campo "Seletor CSS" no painel ⚙ da conta.
Ordem de prioridade dos seletores
Tipo Exemplo Estabilidade
ID — atributo id #mfaCode Ótima
Name — atributo name input[name="otp"] Ótima
Autocomplete semântico input[autocomplete="one-time-code"] Boa
Placeholder input[placeholder="enter code"] Razoável
Classe CSS .mfa-input Frágil
Como ler o HTML inspecionado

Exemplo de elemento que você verá no DevTools:

<input id="mfaCode" ← use #mfaCode name="mfaCode" ← ou input[name="mfaCode"] autocomplete="one-time-code" type="text" placeholder="enter code" >

Neste caso use #mfaCode — o id é o atributo mais estável.

3 Input único ou campos separados?

Alguns sites usam um único campo para os 6 dígitos. Outros mostram 6 caixinhas separadas (uma por dígito). A extensão detecta automaticamente — mas o seletor muda.

✓ Input único
<input id="app_totp" maxlength="6">

Seletor: #app_totp
Retorna 1 elemento → preenche o código inteiro.

⬡ 6 campos separados
<input name="code-0" maxlength="1"> <input name="code-1" maxlength="1"> ... (até code-5)

Seletor: input[name^="code-"]
Retorna 6 elementos → distribui 1 dígito por campo.

💡 O ^= no seletor significa "começa com". Use quando os campos têm um prefixo em comum no atributo name ou id.

5 Alternativa: use a IA para identificar

Se não quiser inspecionar manualmente, copie o HTML do campo e use o prompt abaixo. A IA retorna tudo pronto para colar na extensão.

1 — Copie o HTML do campo MFA
1
Abra a página de MFA do site e deixe o campo de código visível.
2
Clique com o botão direito no campo (ou na área ao redor) → Inspecionar.
3
No DevTools, o elemento <input> fica destacado em azul. Clique com o botão direito nele → Copy → Copy outerHTML.
4
Se o campo fizer parte de um formulário maior, você também pode copiar o elemento pai (<form> ou <div> que o contém) para dar mais contexto à IA.
2 — Use este prompt
Estou configurando minha extensão MFA Vault (Chrome) e preciso identificar os dados para preencher automaticamente o código MFA neste site. Site: [NOME DO SITE] URL da página de MFA: [COLE A URL AQUI] HTML do campo (copiado via DevTools → Inspecionar → Copy outerHTML): [COLE O HTML AQUI] Por favor, retorne exatamente neste formato: URL Pattern: (trecho da URL que identifica a página, ex: github.com) Seletor CSS: (seletor mais estável do campo de input, preferir id > name > atributos semânticos) Tipo: (Input único OU 6 campos separados) autoSubmit: (true se houver botão de envio no mesmo formulário, false se não tiver certeza)
3 — Aplique o resultado na extensão
1
Abra a extensão MFA Vault e clique em ⚙ na conta correspondente ao site.
2
Clique em + Adicionar regra (ou ↙ Site conhecido se já existir um preset).
3
Preencha URL Pattern e Seletor CSS com os valores retornados pela IA.
4
Marque "Enviar formulário após preencher" se o autoSubmit for true.
5
Clique fora para salvar e teste clicando no código com o site aberto.
💡 Se aparecer "Campo não encontrado — copiado", o seletor pode estar errado ou a página ainda não renderizou o campo. Tente copiar um trecho maior de HTML (o <form> ou o componente pai) e repita o processo.
